MISSION 2

Objectif

Cette mission a pour but d'exploiter la base de données de KDou pour générerdes analyses et des indicateurs stratégiques. Les données présentes, qu'elles soient opérationnelles, financières ou commerciales, offrent une perspective précieuse pour comprendre les mécanismes internes de l'entreprise et déceler des pistes d'optimisation.

Outil Utilisé : Low Code Access

Nous avons choisi le logiciel Low Code Access afin de faciliter le développement de requêtes et l’extraction d’informations. Cet outil intuitif permet à des utilisateurs non expérimentés en programmation de créer des requêtes rapidement et de visualiser les données de manière efficace, tout en minimisant les erreurs potentielles.

Exemples de Requêtes et Résultats

Requête QBE :
SELECT Fournisseur.Contact FROM Fournisseur WHERE Fournisseur.Contact="Marie Delamare" ORDER BY Fournisseur.Contact;

Résultat :
Nous en déduisons que la fournisseuse Marie Delamare n’existe pas dans la base de données.

Requête QBE :
SELECT Fournisseur.Societe, Fournisseur.Fonction FROM Fournisseur WHERE Fournisseur.Societe="Ma Maison" ORDER BY Fournisseur.Societe, Fournisseur.Fonction;

Résultat :
Le fournisseur “Ma Maison” a pour fonction “Directeur du marketing”.

Requête QBE :
SELECT Fournisseur.Pays, Categorie.NomCateg FROM Fournisseur, Categorie WHERE Fournisseur.Pays="Singapour" ORDER BY Categorie.NomCateg;

Résultat :
Les catégories de produits incluent boissons, condiments, desserts, pâtes, poissons, produits laitiers, produits secs et viandes.

Requête QBE :
SELECT DISTINCTROW Categorie.NomCateg, Produit.NomProd FROM (Client INNER JOIN Commande ON Client.CodeCli = Commande.CodeCli) INNER JOIN ((Categorie INNER JOIN Produit ON Categorie.CodeCateg = Produit.CodeCateg) INNER JOIN DetailCommande ON Produit.RefProd = DetailCommande.RefProd) ON Commande.NoCom = DetailCommande.NoCom WHERE Categorie.NomCateg="Viandes" AND Client.VilleCli="Paris" ORDER BY Produit.NomProd;

Résultat :
Les produits de type viandes livrés à Paris incluent : “Alice Mutton”, “Mishi Kobe Niku”, etc.

Analyse des Données

Le chiffre d’affaires total est de 66 344,82€. De plus, aucune des clients n’a été identifiée comme n’ayant pas passé de commande.

Problèmes Rencontrés

Nous avons rencontré des difficultés de compréhension dues à la formulation des demandes et à la complexité de certaines requêtes. Cela a affecté notre capacité à interpréter correctement les résultats.

Améliorations Potentielles

Une clarification des attributs, notamment ceux avec des noms abrégés, serait bénéfique. De plus, il serait judicieux de corriger les incohérences, comme l’écriture du pays “Suède” qui apparaît de deux manières différentes.
